# ТЗ: Арена агентов (multi-agent self-play)

> Несколько наших AI-персон в одной реальной TG-группе общаются между собой;
> одобренные модератором реплики становятся обучающим корпусом (стиль-банк).
> Согласовано с Bruno 2026-05-26.

---

## 0. Цель и согласованные решения

| Вопрос                         | Решение                                                                              |
| ------------------------------ | ------------------------------------------------------------------------------------ |
| **Главная цель**               | Генерация обучающего корпуса (диалоги для self-learning агента)                      |
| **Площадка**                   | Реальная приватная TG-группа (MTProto), НЕ виртуальный сэндбокс                      |
| **Кто создаёт группу**         | Bruno создаёт сам, вводит `chat_id`/ссылку — арена работает на готовой               |
| **Режим общения**              | Настраиваемый при создании: `casual` (живая болтовня) \| `sales` (тренировка продаж) |
| **Сбор корпуса**               | Только одобренные модератором 👍 реплики идут в обучающий пул                        |
| **Реальные данные (анти-эхо)** | Пока нет — арена первый источник ⇒ усиленные анти-эхо меры обязательны               |

### ⚠️ Известное ограничение (принято осознанно)

Обучение агента на разговорах агентов же = **эхо-камера / model collapse**: модели
усиливают свои шаблоны, а не становятся «человечнее». Для очеловечивания реальные
диалоги людей в разы ценнее. Митигации (раздел 5) снижают риск, но НЕ убирают его.
**Рекомендация:** как можно раньше добавить скрейп реальных тематических чатов как
основной источник стиля, а арену использовать для (а) тренировки продаж, (б) отладки
промптов/оценки персон, (в) лёгкого seed-корпуса.

## 1. Что переиспользуем (уже в коде)

- **Персоны** `tg_ai_personas` — system_prompt, account_ids, niche, ai_model, temperature, schedule, RAG.
- **Агент-луп** `app/tasks/ai_agent_tasks.py::_ai_agent_async` — коннект аккаунта, чтение контекста чата, генерация, отправка через Telethon, анти-бан (active hours, задержки, FloodWait, детект бана, авто-пауза + notify).
- **`_generate_opener`** — реплика по теме чата БЕЗ RAG/рекламы → база для casual-режима.
- **`_humanize_text`** — анти-робот (убирает тире/штампы).
- **Стиль-банк** `tg_style_samples` + `app/api/v1/style_bank.py` + `_fetch_style_examples`/`_build_style_block` — few-shot обучение, импорт/анонимизация.
- **Self-learning** `_fetch_feedback_examples`/`_build_feedback_block` — few-shot good/bad по рейтингам.
- **Создание/инвайты/членство** `channel_creator.py`, `channel_tasks.py`, humanity-gate, membership-check.
- **Живучесть лупа** reaper (beat + worker_ready).

## 2. Данные

### Новая таблица `tg_agent_arenas`

```sql
CREATE TABLE IF NOT EXISTS tg_agent_arenas (
  id            TEXT PRIMARY KEY,
  name          TEXT NOT NULL,
  chat_id       TEXT,                 -- TG-группа (вводит Bruno: id/@username/ссылка → резолв)
  chat_title    TEXT,
  persona_ids   TEXT DEFAULT '[]',    -- JSON: 2–4 персоны-участника
  mode          TEXT DEFAULT 'casual',-- casual | sales
  topic         TEXT,                 -- тема/сценарий-затравка
  sales_config  TEXT,                 -- JSON (для sales): кто продаёт, кто лиды, возражения
  cadence_sec   INTEGER DEFAULT 120,  -- пауза между ходами (тиками)
  max_msgs_day  INTEGER DEFAULT 100,  -- дневной лимит сообщений на арену
  turn_order    TEXT DEFAULT '[]',    -- JSON: порядок ходов персон (round-robin)
  next_turn_idx INTEGER DEFAULT 0,    -- чей ход
  status        TEXT DEFAULT 'DRAFT', -- DRAFT | RUNNING | PAUSED | STOPPED
  total_msgs    INTEGER DEFAULT 0,
  total_harvested INTEGER DEFAULT 0,  -- сколько сниппетов одобрено в корпус
  created_at    TEXT DEFAULT (datetime('now')),
  updated_at    TEXT DEFAULT (datetime('now'))
);
```

### Транскрипт — переиспользуем `tg_ai_messages`

Сообщения арены логируем туда же (`persona_id`, `chat_id`, `ai_text`, `original_text`,
`moderator_rating`), помечая `chat_id` группы арены + признак арены в metadata/отдельном
поле `arena_id` (добавить колонку миграцией). `moderator_rating='good'` = кандидат в корпус.

### Корпус — `tg_style_samples` (есть)

Добавляем `source='arena'` к сниппетам. Колонка `source` уже есть.

## 3. Провижн группы (Bruno создаёт)

1. Bruno создаёт приватную супергруппу в Telegram, добавляет в неё аккаунты персон.
2. В UI арены вводит `chat_id`/`@username`/ссылку → сервис резолвит и проверяет, что
   все персоны-аккаунты состоят в группе (membership-check). Недостающих — подсветить.
3. (v2, опц.) авто-создание группы + инвайты через аккаунт (готовые кирпичи есть).

---

## 4. Цикл диалога — Celery-таск `arena_tick(arena_id)`

По расписанию (`cadence_sec`, в рамках active hours и `max_msgs_day`):

1. Берём арену, если `status=RUNNING`.
2. Выбираем персону по `turn_order[next_turn_idx]` (round-robin) + джиттер, чтобы
   3 бота не писали залпом.
3. Коннектим её аккаунт (NO_PROXY-гейт, прокси обязателен).
4. Читаем последние N сообщений группы (контекст).
5. Генерируем реплику:
   - **casual:** характер персоны + few-shot стиль-банк + тема, БЕЗ Atlas/рекламы
     (логика `_generate_opener`/реплай, `_humanize_text`).
   - **sales:** роль из `sales_config` — «продавец» ведёт по скрипту (RAG Atlas разрешён),
     «лиды» отыгрывают возражения.
6. `_human_pause` (печатает) → отправка через Telethon.
7. Лог в `tg_ai_messages` (`arena_id`, `moderator_rating=NULL`).
8. `next_turn_idx=(next_turn_idx+1)%len`, `total_msgs++`.
9. Анти-бан: FloodWait → пауза; детект бана/кика → `status=PAUSED` + notify_admin.

Запуск: beat-задача дергает активные арены раз в `cadence`, либо self-reschedule
(как ai_agent loop) под reaper.

---

## 5. Сбор корпуса — `arena_harvest(arena_id)` + модерация

1. Берём сообщения арены из `tg_ai_messages` за период.
2. Нарезаем на сниппеты 2–6 реплик (по reply-цепочкам/последовательности).
3. **Анонимизация:** имена → Имя1/Имя2/…, вырезать @username, ссылки, телефоны, любые PII.
4. Дедуп коротких сниппетов.
5. Показываем в UI на **модерацию** (👍/👎).
6. **Только 👍** → запись в `tg_style_samples` (`source='arena'`, `topic=arena.topic`),
   `total_harvested++`.

### Анти-эхо-камера (обязательно)

- Арена-сниппеты помечены `source='arena'`; в few-shot пуле их доля **ограничена**
  (напр. ≤30%), остальное — реальные данные (когда появятся).
- Жёсткий гейт качества = ручная модерация 👍 (выбор Bruno).
- (план) скрейп реальных чатов как основной источник + seed-затравки в группу.
- Метрика разнообразия: предупреждать, если арена-стиль доминирует в пуле.

## 6. UI — вкладка «Арена»

- Список арен (статус, режим, сообщений, собрано в корпус).
- Создание: имя, группа (chat_id/ссылка → резолв+проверка членства), 2–4 персоны,
  режим (casual/sales + sales_config), тема, темп, дневной лимит.
- Карточка арены: **живой транскрипт** (кто/что/время), Старт/Пауза/Стоп, статистика
  (сообщений, стоимость $, собрано сниппетов), кнопка «Собрать корпус».
- **Модерация сниппетов:** очередь сниппетов с 👍/👎 → одобренные в стиль-банк.

---

## 7. Гайдрейлы

- budget-check (общий месячный лимит AI) перед генерацией.
- Только приватная группа; агенты НЕ пишут друг другу в ЛС.
- Детект бана/кика → пауза арены + notify_admin.
- FloodWait → пауза аккаунта.
- Лимиты per-account per-day соблюдаются (общие настройки).
- Прокси обязателен (NO_PROXY-гейт).
