# Телеграм Сервис — Руководство пользователя

Пошаговая инструкция для работы с TG Service.

---

## 1. Запуск сервиса

### Что нужно перед стартом

- Python 3.12 + виртуальное окружение (`.venv/`)
- Redis (для Celery очередей)
- Файл `.env` с настройками (скопируй из `.env.example`)

### Запуск через PM2

```bash
cd tools/tg-service
pm2 start ecosystem.config.js
```

Это запустит 3 процесса:

- **pup-tg-api** — веб-сервер (порт 8001)
- **pup-tg-worker** — фоновые задачи (Celery)
- **pup-tg-beat** — планировщик (Celery Beat)

### Ручной запуск (для разработки)

```bash
# Терминал 1 — API
.venv/bin/uvicorn app.main:app --host 127.0.0.1 --port 8001 --reload

# Терминал 2 — Worker
.venv/bin/celery -A app.tasks.celery_app worker -Q pup_tg_default -c 4 --loglevel=info

# Терминал 3 — Beat (опционально)
.venv/bin/celery -A app.tasks.celery_app beat --loglevel=info
```

### Открыть в браузере

```
http://localhost:8001
```

## 2. Первоначальная настройка

### 2.1 Настройки (шестерёнка внизу сайдбара)

Перейди в **Настройки** и заполни:

**Telegram API (обязательно):**

- `App ID` и `App Hash` — получаешь на https://my.telegram.org
- Без этого аккаунты не смогут подключаться

**AI (для работы агентов):**

- `Anthropic API Key` — ключ от Claude API
- `Модель по умолчанию` — рекомендую Haiku 4.5 (дёшево и быстро)
- `Лимит бюджета` — месячный лимит в USD

**Дневные лимиты (anti-ban):**

- DM в день: 30 (рекомендовано)
- Постов в чатах: 3
- Комментариев: 10
- Инвайтов: 180

**Активные часы:**

- По умолчанию 09:00-22:00 — аккаунты не работают ночью

### 2.2 Добавление прокси

Перейди в **Прокси** → кнопка **"Добавить"**.

Формат ввода (один прокси на строку):

```
host:port:login:password
```

Пример:

```
123.45.67.89:1080:myuser:mypass
```

Настройки при добавлении:

- **Тип:** RESIDENTIAL (рекомендуется для TG)
- **Схема:** HTTP или SOCKS5
- **Страна:** укажи страну прокси

После добавления нажми **"Проверить все"** — система протестирует каждый прокси и покажет статус + пинг.

### 2.3 Импорт Telegram-аккаунтов

Перейди в **Аккаунты** → кнопка **"Импорт из ZIP"**.

Подготовь ZIP-архив с файлами:

```
accounts.zip
├── +79161234567.session     # файл сессии Telethon
├── +79161234567.json        # метаданные аккаунта
├── +79312345678.session
├── +79312345678.json
└── ...
```

JSON файл содержит:

```json
{
  "app_id": 12345678,
  "app_hash": "abcdef1234567890",
  "twoFA": "пароль_2fa_если_есть"
}
```

После импорта:

1. Аккаунты появятся в списке со статусом **IMPORTED**
2. Привяжи прокси к каждому аккаунту (автоназначение или вручную)
3. Нажми **"Чек Telegram"** на каждом аккаунте — это проверит:
   - Авторизацию
   - Может ли отправлять ЛС
   - Может ли искать пользователей
   - Есть ли ограничения (shadow ban)
4. Рабочие аккаунты получат статус **ACTIVE**

## 3. База знаний (для AI-агентов)

Прежде чем запускать AI-агентов, загрузи базу знаний о проекте.

### Через API (пока нет экрана в UI):

```bash
curl -X POST "http://localhost:8001/api/v1/kb/documents?workspace=default" \
  -H "x-admin-token: ваш-токен" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "О проекте Atlas System",
    "content": "Здесь текст описания проекта... Что это, как работает, преимущества, реферальная программа..."
  }'
```

Система автоматически разобьёт текст на чанки по 500 символов. AI-агенты будут использовать эти знания при общении.

Запомни `id` документа — он понадобится при создании персоны.

## 4. Парсинг аудитории

### 4.1 Создание задачи парсинга

Перейди в **Парсер аудитории** → **"Создать задачу"**.

Заполни:

- **Название:** например "Крипто чаты"
- **Режим парсинга:**
  - `CHAT_MEMBERS` — все участники группы (если открыты)
  - `WRITERS` — авторы сообщений в чате
  - `COMMENTERS` — авторы комментариев к постам
  - `REACTIONS` — кто ставил реакции на посты
  - `JOINERS` — кто недавно вступил
  - `TOPICS` — участники тем форума
  - `GLOBAL_SEARCH` — поиск по ключевым словам
- **Источники:** список чатов/каналов (по одному на строку):
  ```
  @crypto_chat
  @web3_community
  https://t.me/defi_group
  ```
- **Фильтры:** только с аватаркой, только с username, без ботов, только Premium

Нажми **"Старт"** — парсинг запустится в фоне. Прогресс отображается в реальном времени.

### 4.2 Просмотр аудитории

Результаты сохраняются в **Базы аудитории**. Там можно:

- Посмотреть список спарсенных пользователей
- Отфильтровать по score, premium, наличию аватарки
- Использовать базу для DM-рассылок, инвайтинга

## 5. Вступление в чаты

Перейди в API (пока функция через UI в разработке):

```bash
curl -X POST "http://localhost:8001/api/v1/join-chats?workspace=default" \
  -H "x-admin-token: ваш-токен" \
  -H "Content-Type: application/json" \
  -d '{
    "target_chats": ["@crypto_chat", "@web3_talks", "https://t.me/+AbCdEfG123"],
    "account_ids": ["id-аккаунта-1", "id-аккаунта-2"],
    "join_interval_min": 30,
    "join_interval_max": 120
  }'
```

Что произойдёт:

1. Каждый аккаунт вступит в каждый чат
2. Между вступлениями — случайная пауза 30-120 секунд
3. После вступления система прочитает описание чата
4. Поддерживаются invite-ссылки (`t.me/+...`) и публичные username

Проверить статус:

```bash
curl "http://localhost:8001/api/v1/join-chats/{task_id}?workspace=default" \
  -H "x-admin-token: ваш-токен"
```

## 6. AI-агент (главная фича)

### 6.1 Создание персоны

Перейди в **AI Промоутер** → **"Создать персону"**.

Заполни:

- **Имя:** "Денис — крипто эксперт"
- **Ниша:** "Web3, DeFi, пассивный доход"
- **Био:** "Трейдер с 5-летним опытом, занимаюсь DeFi стратегиями"
- **Личность:** "Дружелюбный, экспертный, не навязчивый"
- **Стратегия:**
  - `soft` — мягко, через экспертизу и помощь
  - `medium` — активнее предлагает
  - `aggressive` — прямые призывы к действию
- **System prompt:** подробная инструкция как вести себя в чатах
- **Аккаунты:** выбери 1-5 аккаунтов для этой персоны
- **Целевые чаты:** список чатов где агент будет работать
- **База знаний:** ID документов из раздела 3
- **DM включён:** да — агент будет писать в ЛС заинтересованным
- **Глубина контекста:** 50-100 сообщений (сколько читать для понимания чата)

### 6.2 Пример system prompt

```
Ты — Денис, опытный крипто-трейдер.

ПРАВИЛА ПОВЕДЕНИЯ В ЧАТАХ:
1. Читай контекст чата перед тем как написать
2. Не рекламируй ничего напрямую
3. Отвечай на вопросы по теме крипты, DeFi
4. Если кто-то спрашивает про пассивный доход — расскажи про свой опыт
5. Говори естественно, с ошибками, как реальный человек
6. Не пиши длинные тексты — 2-3 предложения максимум
7. Используй эмодзи умеренно

ПРАВИЛА ЛС:
1. Если человек заинтересовался — пиши ему лично
2. В ЛС расскажи подробнее о проекте
3. Дай реферальную ссылку когда человек готов
4. Не давай ссылку в первом же сообщении
```

### 6.3 Активация

Нажми **"Активировать"** на персоне. Система запустит агента который:

1. **Читает чаты** — последние 50-100 сообщений + описание чата
2. **Отвечает в чатах** — на подходящие сообщения, с учётом базы знаний
3. **Оценивает интерес** — Claude ставит оценку 0-10 каждому пользователю
4. **Пишет в ЛС** — если оценка >= 7, начинает personal sales-диалог
5. **Ведёт диалог в ЛС** — с базой знаний, проводит через этапы воронки

Между сообщениями — паузы 1-5 минут (anti-ban).

## 7. DM-рассылка

### Подготовка

1. Спарси аудиторию (раздел 4)
2. Создай шаблон сообщения в **Шаблоны** (можно несколько вариантов для A/B)

### Запуск

Перейди в **Рассылка в ЛС** → **"Создать кампанию"**:

- **Аудитория:** выбери спарсенную базу
- **Шаблон:** выбери шаблон
- **Аккаунты:** какие аккаунты будут отправлять
- **Задержка:** 30-180 секунд между сообщениями

Нажми **"Старт"**. Система:

- Отправляет сообщения с рандомной задержкой
- Чередует варианты шаблона (A/B)
- Подставляет {first_name}, {username} в текст
- Останавливается автоматически при FloodWait или >30% ошибок
- Показывает прогресс в реальном времени

- **Целевые чаты:** список групп для постинга

## 10. Скрипты прогрева

Прогрев делает аккаунты "живыми" — имитирует поведение реального человека.

### Создание скрипта

Через API:

```bash
curl -X POST "http://localhost:8001/api/v1/warmup-scripts/scripts?workspace=default" \
  -H "x-admin-token: ваш-токен" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Утренний прогрев",
    "description": "Базовая активность для холодных аккаунтов",
    "target_channels": ["@telegram", "@durov", "@tginfo"],
    "actions": [
      {"type": "subscribe", "count": 2},
      {"type": "react", "count": 5, "emoji": "👍"},
      {"type": "read_chats", "count": 10},
      {"type": "comment", "count": 1, "text": "Интересно!"}
    ]
  }'
```

### Типы действий

| Тип            | Что делает                             |
| -------------- | -------------------------------------- |
| `subscribe`    | Подписка на каналы                     |
| `react`        | Реакции на посты (emoji настраивается) |
| `comment`      | Комментарий под постом                 |
| `read_chats`   | Чтение диалогов (имитация активности)  |
| `view_stories` | Просмотр сторис                        |

### Запуск

```bash
curl -X POST "http://localhost:8001/api/v1/warmup-scripts/scripts/{script_id}/run?workspace=default" \
  -H "x-admin-token: ваш-токен" \
  -H "Content-Type: application/json" \
  -d '{"account_ids": ["id-акка-1", "id-акка-2", "id-акка-3"]}'
```

Система выполнит все действия на каждом аккаунте автономно.

## 12. Мониторинг и безопасность

### Статусы аккаунтов

| Статус       | Значение                                       |
| ------------ | ---------------------------------------------- |
| IMPORTED     | Только импортирован, не проверен               |
| ACTIVE       | Рабочий, готов к использованию                 |
| WARMING      | На прогреве                                    |
| FLOOD_WAIT   | Временная блокировка (автоматически пауза 24ч) |
| SPAM_BLOCKED | Спамблок от Telegram                           |
| BANNED       | Забанен                                        |
| DEAD         | Сессия не валидна                              |

### Экстренная остановка

Система автоматически останавливается если:

- > 30% аккаунтов получили бан
- > 50% сообщений удалены модераторами
- FloodWait >5 минут

### Карточка аккаунта

Нажми на аккаунт → увидишь:

- **Обзор** — статус, прокси, прогрев, лимиты, статистика
- **Активность** — лог действий прогрева
- **Прокси** — детали привязанного прокси
- **Чаты** — диалоги аккаунта (кликабельные → мессенджер)
- **2FA** — пароль 2FA, App ID/Hash, активные сессии Telegram
